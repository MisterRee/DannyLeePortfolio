Premise

Right as a user connects, he or she is given a randomly generated color as identification.

Players fight over claiming as many sporadically appearing white bubbles as they can by passing their mouse over them.

Players must also avoid other player's bubbles or else all of their claimed bubbles revert back to white, unclaimed bubbles.

Execution

Multiple users interact with each other using a websocket server. The server sends packets of simple data to each connected user multiple times a second to keep things animated smoothly.

The node server is used to do all calculations each frame for accuracy through every client. The front-end client only has to draw the streamed data and stream mouse coordinate data back to the websocket.
